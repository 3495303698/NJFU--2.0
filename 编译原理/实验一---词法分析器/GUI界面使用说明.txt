# 词法分析器GUI版本使用说明

## 1. 程序概述

这个词法分析器已经从控制台应用程序升级为图形用户界面(GUI)应用程序，不再需要通过命令提示符运行。程序界面包含了用户要求的三个主要部分：
- 文件名输入区域
- 错误信息输出区域
- 分析结果输出区域

## 2. 编译程序

请按照以下步骤编译程序：

1. 打开**Visual Studio命令提示符**（注意：不是普通的命令提示符）
2. 使用`cd`命令导航到程序所在目录
3. 运行`编译GUI应用.bat`脚本
4. 如果编译成功，会生成`词法分析器.exe`文件

## 3. 运行程序

编译完成后，您可以：
- 直接双击`词法分析器.exe`运行程序
- 或者运行之前创建的`运行词法分析器.bat`脚本

## 4. 界面使用说明

程序窗口分为三个主要区域：

### 4.1 文件名输入区域
- 在"请输入分析的文件名:"文本框中输入要分析的文件名
- 文件名需要包含扩展名，如`program1.txt`
- 如果文件不在程序同一目录下，需要提供完整路径
- 输入完成后，点击"开始词法分析"按钮

### 4.2 错误信息输出区域
- 显示文件打开失败、分析错误等信息
- 如果成功打开文件，会显示分析结果的摘要信息，如识别出的token数量
- 如果没有找到可识别的token，会显示相应提示

### 4.3 分析结果输出区域
- 以表格形式显示分析出的所有token
- 每行包含三个部分：token值、token类型、行号
- 可以滚动查看所有结果

## 5. 常见问题解答

### 5.1 程序闪退问题
本版本通过以下方式解决了闪退问题：
- 使用纯GUI模式编译，避免控制台窗口
- 优化了界面交互逻辑
- 分析完成后不会自动关闭程序

### 5.2 文件无法打开
- 确保输入的文件名正确
- 检查文件是否存在
- 如果文件不在程序目录下，需要提供完整路径

### 5.3 显示的token类型
程序识别以下类型的token：
- 保留字（如main, int, if等）
- 标识符
- 常数（数字和字符串）
- 运算符
- 分隔符

## 6. 示例文件

如果需要测试程序，可以创建一个简单的文本文件，例如`program.txt`，内容如下：

```
main() {
    int a = 10;
    if (a > 5) {
        printf("Hello, World!\n");
    }
    return 0;
}
```

输入该文件名，点击分析按钮查看结果。
