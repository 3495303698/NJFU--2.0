# 实验二：语法分析法1 - 递归子程序法

## 项目简介

本项目实现了基于递归下降分析法的语法分析程序，用于分析给定的表达式是否满足以下文法规则。

## 文法规则

```
(1) E -> TG
(2) G -> +TG | -TG
(3) G -> ε
(4) T -> FS
(5) S -> *FS | /FS
(6) S -> ε
(7) F -> (E)
(8) F -> i
```

其中：
- `E`, `G`, `T`, `S`, `F` 为非终结符
- `+`, `-`, `*`, `/`, `(`, `)`, `i`, `#` 为终结符
- `ε` 表示空串
- `#` 为输入结束符

## 编译方法

### 使用 g++ 编译

```bash
g++ -o parser.exe src/main.cpp src/parser.cpp -I include
```

### 使用 Microsoft Visual C++ 编译

```bash
cl /EHsc src/main.cpp src/parser.cpp /I include /Fe:parser.exe
```

## 使用方法

1. 编译程序（见上述编译方法）
2. 运行 `parser.exe`
3. 按照提示输入以 `#` 结束的符号串
4. 程序将输出该符号串是否为合法符号串

## 输入输出示例

### 合法符号串示例

- `i+i#`
- `i-i#`
- `i*i#`
- `i/i#`
- `i+i*i#`
- `(i+i)*i#`
- `i*(i+i)#`

### 非法符号串示例

- `i+i*#` （运算符后缺少操作数）
- `i++#` （连续运算符）
- `(i+i#` （括号不匹配）
- `i#` （虽然单个i合法，但需根据上下文判断）

## 功能特点

1. **递归下降分析**：为每个非终结符实现对应的递归函数
2. **错误检测**：能够检测并报告详细的错误信息，包括错误位置和错误类型
3. **词法分析**：内置词法分析器，识别运算符、括号、标识符等
4. **友好界面**：提供清晰的提示信息和结果输出

## 程序思路

1. **词法分析**：`nextToken()` 函数负责从输入字符串中提取下一个词法单元
2. **语法分析**：为每个非终结符（E, G, T, S, F）实现对应的递归函数
3. **错误处理**：当遇到不符合文法规则的输入时，记录错误信息并停止分析
4. **主程序**：处理输入输出，调用分析器并显示结果

## 注意事项

1. 输入必须以 `#` 结束
2. 表达式中允许使用运算符（+-*/）、分隔符（括号）、字符 i、结束符 #
3. 如果遇到错误的表达式，程序会输出详细的错误提示信息
4. 程序会跳过空白字符（空格、制表符、换行符）
