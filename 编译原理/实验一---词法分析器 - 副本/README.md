# 词法分析器项目

这是一个C语言子集的词法分析器实现，用于编译原理实验一。

## 功能特性

- 识别保留字（关键字）：main, printf, if, int, for, while, do, return, break, continue
- 识别标识符（变量名、函数名等）
- 识别无符号整数常量
- 识别字符串常量
- 识别运算符：+, -, *, /, =, >, <, >=, <=, !=
- 识别分隔符：, ; { } ( )
- 输出Token的二元组格式：(类型, 单词)
- 统计错误字符个数

## Token类型编码

- **1**: 保留字（关键字）
- **2**: 标识符
- **3**: 常数（整数、字符串）
- **4**: 运算符
- **5**: 分隔符

## 编译方法

### 使用CMake（推荐）

```bash
mkdir build
cd build
cmake ..
cmake --build .
```

### 使用g++直接编译

```bash
g++ -std=c++11 main.cpp lexer.cpp -o 词法分析器
```

### 使用Visual Studio

1. 创建新的控制台应用程序项目
2. 将 `main.cpp`, `lexer.cpp`, `lexer.h` 添加到项目
3. 编译运行

## 使用方法

### 命令行方式

```bash
# 基本用法
./词法分析器 program1.txt

# 指定输出文件
./词法分析器 program1.txt output.txt
```

### 交互式方式

直接运行程序，然后按提示输入文件名：

```bash
./词法分析器
请输入要分析的文件名: program1.txt
```

## 输出格式

程序会输出每个Token的二元组格式：

```
(1, "main")
(5, "(")
(5, ")")
(5, "{")
(1, "int")
(2, "i")
(5, ",")
...
```

同时会生成 `output.txt` 文件保存分析结果。

## 程序要求

根据实验要求：

1. 源程序采用自由书写格式（一行可多个语句，一个语句可多行）
2. 标识符前20个字符有效
3. 整数用2个字节表示
4. 长整数用4个字节表示
5. 删除空格类字符（包括回车、制表符、空格）
6. 按拼写单词，用(内码, 属性)二元式表示
7. 根据需要填写标识符表供后续阶段使用

## 设计说明

### 核心类：Lexer

- `loadFromFile()`: 从文件加载源程序
- `getNextToken()`: 获取下一个Token
- `hasMoreTokens()`: 判断是否还有更多Token
- `getErrorCount()`: 获取错误字符计数

### 识别流程

1. 初始化：加载源程序到内存
2. 跳过空白字符
3. 根据首字符判断Token类型：
   - 字母开头 → 标识符或保留字
   - 数字开头 → 数字常量
   - 引号开头 → 字符串常量
   - 运算符字符 → 运算符
   - 分隔符字符 → 分隔符
4. 输出Token二元组

## 注意事项

- 确保测试文件（.txt格式）放在程序运行目录下
- 程序会自动跳过空白字符
- 未识别的字符会被计入错误计数
- 标识符超过20个字符会被截断
